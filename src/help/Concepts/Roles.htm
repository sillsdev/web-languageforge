Roles

Roles determine what a person can see or do in Language Forge. They are hierarchical in the sense that they can have decreasing numbers of enabled tasks until the last role, which has the fewest enabled tasks. It is expected that separate language projects will have differences regarding how roles and tasks are arranged.

Here is an overview of how roles could be set up in a project.

  • Manager and Project Owner

For the person in this role, all tasks that can be done are enabled. Typically, they enable tasks for individuals assigned to other roles. Also, they typically initiate Send/Receive to update the repository in Language Depot.

See: Manager and Project Owner tasks overview

  • Manager

The person in this role can do any of the tasks that a Manager and Project Owner can do, except delete a project. However, unlike the Manager and Project Owner, it is possible to reduce the number of tasks a Manager can do.

See: Manager tasks overview

  • Contributor

The person in this role can typically do anything an Observer with comment can do. In addition, they can work with the lexical data, such as add lexical entries.

See: Contributor tasks overview

  • Observer with comment

The person in this role can open and look at language projects. In addition, they can work with comments.

See: Observer with comment tasks overview

  • Observer

The person in this role can only open and look at language projects.

See: Observer tasks overview
